# 制作前に決まったこと

AIテクスチャジェネレーターの初期プロトタイプで採用する仕様と判断をまとめます。

## サービスの目的

外部AIが生成したMinecraft向け16x16テクスチャJSONを、ブラウザ上で読み込み、プレビュー、編集し、PNG画像と更新済みJSONとして保存できるようにします。

## 対象テクスチャ

- ブロック用テクスチャとアイテム用テクスチャの両方を扱います。
- ピクセルデータの構造はブロック用とアイテム用で共通です。
- Minecraftのリソースパック構造に合わせ、サービス上では `block` と `item` を区別します。
- 想定配置先は次の通りです。

```text
assets/minecraft/textures/block/{texture_name}.png
assets/minecraft/textures/item/{texture_name}.png
```

## JSON仕様

JSON仕様は [json_format_spec.md](./json_format_spec.md) にまとめます。

- `resolution` は `{ "width": 16, "height": 16 }` の数値形式に統一します。
- `texture_type` は `"block"` または `"item"` を使います。
- `pixel_data` は16行x16列の2次元配列です。
- 各ピクセルは `#RRGGBB` または `transparent` のみ許可します。
- 色数制限は初期プロトタイプでは設けません。

## テクスチャ名

`texture_name` はMinecraftのファイル名として使える文字に制限します。

```text
^[a-z0-9_]+$
```

許可する文字:

- 英小文字
- 数字
- アンダースコア

許可しない文字:

- 英大文字
- 日本語
- スペース
- ハイフン
- ドット
- スラッシュ
- その他の記号

不正な名前は自動変換せず、エラーとして表示します。

## 保存

- PNG保存時は `{texture_name}.png` を提案ファイル名にします。
- JSON保存時は `{texture_name}.json` を提案ファイル名にします。
- 保存先選択ダイアログが出るかどうかは、OSではなく主にブラウザのダウンロード設定に従います。
- PNGは透明ピクセルを保持します。

## 編集

- 16x16のキャンバスを拡大表示します。
- カラーピッカーで色を選び、キャンバスをクリックまたはドラッグして塗ります。
- 消しゴムは対象ピクセルを `transparent` にします。
- JSON出力はプロトタイプでは「JSONを書き出し」ボタンで更新します。

## エラー対応

JSONが不正な場合は、画面上にエラー内容を表示します。
同時に、外部AIへそのまま送れる修正依頼文も生成します。

修正依頼文には次を含めます。

- エラー内容
- 正しいJSON仕様の要点
- 修正対象の元JSON
- JSONのみで出力する指示

## カスタムGem用プロンプト

- 初期プロトタイプの画面内に、GeminiのカスタムGemへ貼り付ける想定のプロンプトを表示します。
- Gemがまだ共有されていない状態でも、ユーザーが自分でプロンプトをコピーして試せるようにします。
- プロンプトの内容は `gemini_prompt.md` と同じ仕様に合わせます。

## デザイン方針

- Minecraftのトーンを参考にします。
- ただし完全なゲームUIには寄せすぎず、制作サービスとして使いやすい見た目にします。
- Stitchで後からデザインを作る前提なので、初期版はプロトタイプ感を残します。
- UI構造は差し替えやすいように、HTML、CSS、JavaScriptを分けます。
