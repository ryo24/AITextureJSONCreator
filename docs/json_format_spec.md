# AIテクスチャJSON仕様

このドキュメントは、AIテクスチャジェネレーターが読み書きするJSON形式を定義します。
対象はMinecraft向けの16x16ピクセルテクスチャです。

## 基本方針

- JSONは1テクスチャにつき1オブジェクトです。
- ピクセルデータは16行x16列の2次元配列で表します。
- 各ピクセルは `#RRGGBB` 形式の色コード、または `transparent` で表します。
- ブロック用テクスチャとアイテム用テクスチャの両方を扱います。
- ピクセル構造はブロックとアイテムで共通です。
- Minecraftのリソースパック構造に合わせるため、サービス上ではブロック用とアイテム用を区別します。
- ブロック用PNGは `assets/minecraft/textures/block/` 配下、アイテム用PNGは `assets/minecraft/textures/item/` 配下に置く想定です。
- アイテム用テクスチャでは透明背景が重要なため、`transparent` を正式なピクセル値として扱います。
- 色数制限は設けません。
- 外部AIが出力しやすく、ブラウザ側で検証しやすい構造を優先します。
- JSON内にコメント、末尾カンマ、省略記号、Markdownのコードフェンスは含めません。

## ルート構造

```json
{
  "schema_version": "1.0",
  "texture_name": "example_texture",
  "texture_type": "block",
  "resolution": {
    "width": 16,
    "height": 16
  },
  "pixel_data": []
}
```

## フィールド

| フィールド | 型 | 必須 | 説明 |
| --- | --- | --- | --- |
| `schema_version` | string | 必須 | JSON仕様のバージョン。初期値は `"1.0"`。 |
| `texture_name` | string | 必須 | テクスチャ名。PNGファイル名にも使う。 |
| `texture_type` | string | 必須 | `"block"` または `"item"`。Minecraft内の保存先カテゴリを決める。 |
| `resolution` | object | 必須 | 画像サイズ。現時点では `width: 16`, `height: 16` のみ対応。 |
| `resolution.width` | number | 必須 | 横ピクセル数。必ず `16`。 |
| `resolution.height` | number | 必須 | 縦ピクセル数。必ず `16`。 |
| `pixel_data` | string[][] | 必須 | 16行x16列のピクセル配列。 |

## ピクセル値

ピクセル値は次のどちらかのみ許可します。

- `#RRGGBB`: 6桁の16進カラーコード。例: `#3FA7D6`
- `transparent`: 透明ピクセル。

許可しない例:

- `#RGB`
- `rgb(255, 0, 0)`
- `rgba(255, 0, 0, 0.5)`
- `none`
- `null`
- 空文字

## テクスチャ名

`texture_name` はMinecraftのファイル名として使える安全な文字だけを許可します。

許可する文字:

- 英小文字 `a` から `z`
- 数字 `0` から `9`
- アンダースコア `_`

許可しない文字:

- 英大文字
- 日本語
- スペース
- ハイフン `-`
- ドット `.`
- スラッシュ `/`
- その他の記号

正規表現:

```text
^[a-z0-9_]+$
```

例:

- OK: `stone_brick_blue`
- OK: `magic_sword_01`
- NG: `StoneBrick`
- NG: `magic-sword`
- NG: `魔法の剣`

PNG出力時の推奨ファイル名:

```text
{texture_name}.png
```

サービス上では、画面で `texture_name` と `texture_type` を入力・選択してから保存します。
ブラウザから保存する場合は、`texture_name` をもとにしたPNGファイル名を提案します。
実際の保存先選択ダイアログの表示有無は、ブラウザとOSのダウンロード設定に従います。

Minecraftリソースパック内へ配置する場合の想定パス:

```text
assets/minecraft/textures/block/{texture_name}.png
assets/minecraft/textures/item/{texture_name}.png
```

## 検証ルール

読み込み時は次を検証します。

- JSONとしてパースできること。
- ルートがオブジェクトであること。
- `schema_version` が `"1.0"` であること。
- `texture_name` が空でない文字列であること。
- `texture_name` が `^[a-z0-9_]+$` に一致すること。
- `texture_type` が `"block"` または `"item"` であること。
- `resolution.width` と `resolution.height` がどちらも `16` であること。
- `pixel_data` が配列であること。
- `pixel_data` の行数が16であること。
- 各行が配列であること。
- 各行の列数が16であること。
- 各ピクセル値が `#RRGGBB` または `transparent` であること。

## 出力時の整形

アプリがJSONを書き出すときは、次の形式を推奨します。

- インデントは2スペース。
- 色コードは大文字に正規化する。例: `#ffaa00` は `#FFAA00`。
- `texture_name` は自動変換せず、不正な文字がある場合はエラーとして扱う。
- `pixel_data` は必ず16行に分けて出力する。

## エラー時のAI修正依頼文

入力JSONが不正な場合、アプリはエラー内容だけでなく、外部AIへそのまま送れる修正依頼文を表示します。
修正依頼文には、次の情報を含めます。

- 何が不正だったか。
- 正しいJSON仕様の要点。
- 修正対象の元JSON。
- 出力はJSONのみ、コメントや説明文なしにする指示。

例:

```text
以下のJSONを、AIテクスチャジェネレーターで読み込める形式に修正してください。

エラー内容:
pixel_data の行数が15行です。必ず16行にしてください。

修正ルール:
- JSONは1つのオブジェクトにしてください。
- schema_version は "1.0" にしてください。
- texture_name は英小文字、数字、アンダースコアのみで作ってください。
- texture_type は "block" または "item" にしてください。
- resolution は { "width": 16, "height": 16 } にしてください。
- pixel_data は16行x16列にしてください。
- ピクセル値は "#RRGGBB" または "transparent" のみ使ってください。
- 出力はJSONのみ。Markdownコードブロック、説明文、コメント、省略記号は使わないでください。

修正対象JSON:
{ここに元JSONを貼る}
```

## 既存プロンプトとの互換性メモ

`gemini_prompt.md` もこの仕様に合わせ、`resolution` は `{ "width": 16, "height": 16 }` の数値形式に統一します。
