# AIテクスチャジェネレーター（JSON→画像変換）Webアプリ 実装計画書

ユーザーがChatGPTやGemini等で生成したJSON形式のピクセルデータ（16x16）を、ブラウザ上で読み込み・プレビュー・編集し、マイクラ用のPNG画像および更新されたJSONとして出力できるWebアプリを構築します。

## User Review Required
> [!IMPORTANT]
> いただいたフィードバックをもとに、技術スタックと機能をアップデートしました。こちらで実装（実行フェーズ）に進んでよいか承認をお願いします。

## Proposed Changes

### 開発環境のセットアップ (プロトタイプ構成)
- **ライブラリ不使用（Vanilla構成）**: プロトタイプとして素早くキャンバス編集機能を実現するため、Reactなどのフレームワークや外部ライブラリを極力使用せず、**「ピュアなHTML / CSS / JavaScript」**で構築します。これにより構成がシンプルになり、将来的に別のライブラリへの移行や組み込みも容易になります。
- 初期プロトタイプは `index.html` を直接ブラウザで開ける構成にします。必要になった段階でViteなどのローカルサーバーを追加します。

### 主要機能の実装

#### [NEW] JSON入力機能
- ユーザーがAIから出力されたJSONをペーストできる「入力用テキストエリア」を配置します。
- JSON仕様は `docs/specs/json_format_spec.md` に従います。
- `resolution` は `{ "width": 16, "height": 16 }` の形式に統一します。
- ブロック用とアイテム用の両方を扱うため、`texture_type` として `"block"` または `"item"` を持たせます。
- ピクセル構造はブロックとアイテムで共通です。
- Minecraftのリソースパック構造に合わせ、ブロック用は `assets/minecraft/textures/block/`、アイテム用は `assets/minecraft/textures/item/` を想定します。
- アイテム用では透明背景が重要なため、`transparent` を正式なピクセル値として扱います。

#### [NEW] テクスチャ名入力・検証
- 画面上で `texture_name` を入力できるようにします。
- `texture_name` はMinecraftのファイル名運用に合わせ、英小文字、数字、アンダースコアのみ許可します。
- 正規表現は `^[a-z0-9_]+$` とします。
- 英大文字、日本語、スペース、ハイフン、ドット、スラッシュ、その他の記号はエラーにします。
- 不正な名前は自動変換せず、ユーザーに修正を促します。
- PNG保存時は `{texture_name}.png` をファイル名として提案します。

#### [MODIFY] エラーハンドリング（コピペ用メッセージ機能）
- JSONが不完全（例：15行しかない、構文エラー等）だった場合、単にエラーを表示するだけでなく、**「外部AI（Gemini等）にそのままコピペして修正を促せるプロンプト文」**を生成して画面に表示します。
  - 修正依頼文には、エラー内容、正しいJSON仕様の要点、修正対象の元JSONを含めます。
  - 出力はJSONのみ、Markdownコードブロックや説明文なしにする指示も含めます。

#### [MODIFY] キャンバス描画・編集（ペイント）機能
- JSONデータをHTML5の`<canvas>`に16x16で描画します（拡大表示）。
- **【追加機能】** キャンバス上のピクセルをマウスクリックで直接編集（塗りつぶし）できるようにします。
  - 描画用の色を選択できるカラーピッカー（`<input type="color">`）と、「消しゴム（透明・transparentにする）」機能を用意します。

#### [NEW] 編集後JSONのリアルタイム出力機能
- 画面下部に「出力用テキストエリア」を新規に作成します。
- キャンバス上でピクセルを編集（クリック）したあと、プロトタイプではまず「書き出し」ボタンで、このテキストエリアのJSONデータを最新の16x16データとして書き出します。
- 実装が複雑にならない場合はリアルタイム更新も対応します。

#### [NEW] 画像ダウンロード機能
- 「画像をPNGで保存」ボタンを設置し、キャンバスの状態をそのまま16x16の `.png` ファイルとしてダウンロードさせます。
- ブラウザのダウンロード機能を使い、保存ファイル名として `{texture_name}.png` を提案します。
- 保存先を選ぶ画面が出るかどうかは、Windows/MacのOS差分ではなく、主にブラウザのダウンロード設定に従います。
- PNGは透明ピクセルを保持して出力します。

## Verification Plan
### ローカル環境でのテスト
- `index.html` をブラウザで開き、UIを確認します。
- 完璧な16x16のJSONを貼り付けて、正しくドット絵が表示されるかを検証します。
- 不正なJSONを貼り付けた際に、AIへ送り返す用のコピペ用メッセージが正しく生成されるか検証します。
- キャンバスをクリックして色が変わり、同時に下部の「出力用テキストエリア」のJSONが正しく書き換わるかを検証します。
- PNG画像としてダウンロード（透過付き）できるかを検証します。
